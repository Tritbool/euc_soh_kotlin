# Real Logs Test Directory

This directory is for **integration testing with real EUC log files**.

## Purpose

The `RealLogsIntegrationTest` will automatically run end-to-end analysis on any CSV files placed here, validating:
- CSV parsing (EUC World / WheelLog formats)
- Battery pack inference (Ns, V_nominal)
- Threshold computation
- Alarm detection
- Summary generation

## Usage

1. **Add real CSV log files** from your EUC:
   ```
   euc-soh-core/src/test/resources/real_logs/
     ├── ride_2024_01_15.csv
     ├── ride_2024_01_22.csv
     └── ride_2024_02_03.csv
   ```

2. **Run tests**:
   ```bash
   ./gradlew test
   ```

3. The test will:
   - ✅ **Run full analysis** if CSV files are present
   - ⊘ **Auto-skip** if this folder is empty (won't fail CI)

## What to add?

- **Format**: CSV files from EUC World or WheelLog apps
- **Quantity**: 5-20 logs minimum for meaningful statistics
- **Span**: Logs covering different dates/km for trend detection
- **Quality**: Valid rides with voltage, current, speed, temperature data

## Auto-skip behavior

If this folder is empty or doesn't contain CSV files:
- Tests will print: `⊘ Skipping real logs test: no CSV files found`
- CI will **not fail** — this is intentional
- Once you add files, tests will automatically enable

## Example log sources

- **EUC World**: Export rides as CSV from app settings
- **WheelLog**: Export logs from ride history
- **Your own**: Any compatible CSV with required columns

## Security note

Do **not commit** real logs if they contain:
- GPS coordinates (privacy)
- Personal information
- Sensitive wheel configuration data

For CI/testing purposes, you can:
- Use anonymized logs (strip GPS data)
- Use synthetic logs generated by `SyntheticDataGenerator`
- Keep real logs locally only (add to `.gitignore`)
